这篇博客算是之前文章的一个补充，详细介绍一下printf，这次是一个比较完善完整的POSIX实现。

printf，格式化输出字符串到stdout，看似简单，实则挺麻烦的一个函数。首先先了解一下POSIX规定的printf的全部功能，然后再看具体实现的算法。

是什么

POSIX规定了printf诸多功能，太基础的基本概念这篇就不说了，从格式化说起。

POSIX中描述了五个printf族函数，分别是dprintf，fprintf，printf，snprintf，sprintf，其中dprintf是对标准的扩展，在多数系统中都有实现，INWOX也不例外。

printf将输出内容放到标准输出流stdout上，fprintf将输出内容放到给定命名的输出流上，sprintf将输出内容放到给定地址然后在最后加上’\0’，调用者要确保给定地址的大小足够，snprintf和sprintf功能一致，只是限制了内容大小，而dprintf则是将内容输出到文件，与fprintf不同的是，此函数第一个参数是文件描述符而不是FILE指针。

格式字符串

%[parameter][flags][width][.precision][length]type
格式字符串由0个或多个指令组成：普通字符（不是%）原样复制到输出流，或conversion specifications（以%开头），每个conversion specifications都会获取0个或多个后续的参数，然后以conversion specifier结尾，在这中间有三个可选的标志，分别为：位置参数、对齐相关参数、最小字段宽度、精度、长度。

  • parameter

    1. n$，n是数字，表示第n个参数，这使得一个参数可以输出多次，如果任意一个占位符使用了parameter，那么其他占位符都需要使用，这是posix扩展，不是ISO C标准

      例子

      printf("%3$d %3$#X, %1$d %1$#X, %2$d %2$#X", 1, 10, 100);

  • flags，可以为0个或多个一下标识：

    1. +，总是表示有符号数值的’+’或’-‘号，缺省情况是忽略正数的符号。仅适用于数值类型。

      例子

      printf("%+d %+d %d %d\n", 10, -10, 10, -10);

    2. -，左对齐。缺省情况是右对齐。

      例子

      printf("|%-20d|%20d|\n", 100, 100);

    3. space，使有符号数的输出如果没有正负号或者输出0个字符，则前缀1个空格。如果空格与’+’同时出现，则空格说明符被忽略。

      例子

      printf("|% d|%d|\n", 10, 10);

    4. #，对于’g’与’G’，不删除尾部0以表示精度。对于’f’, ‘F’, ‘e’, ‘E’, ‘g’, ‘G’, 总是输出小数点。对于’o’, ‘x’, ‘X’, 在非0数值前分别输出前缀0, 0x, 0X表示数制。

      例子

      printf("%#G %G\n", 1.0, 1.0);

      printf("%#x %x\n", 10, 10);

    5. 0，如果width选项前缀以0，则在左侧用0填充直至达到宽度要求，如果0与-均出现，则0被忽略，即左对齐依然用空格填充。

      例子

      printf("|%020d|%20d|\n", 1, 1);

      printf("|%020d|%20d|\n", -1, -1);

  • width，可以直接给定也可以忽略，或者使用*指定参数的值

    给出显示数值的最小宽度，实际输出字符的个数不足域宽，则根据左对齐或右对齐进行填充。实际输出字符的个数超过域宽并不引起数值截断，而是显示全部。宽度值的前导0被解释为0填充标志。如果域宽值为*，则由对应的函数参数的值为当前域宽

    例子

    printf("|%20d|%d|\n", 10, 10);

  • precision，可以直接给定也可以忽略，或者使用*指定参数的值

    通常指明输出的最大长度，依赖于特定的格式化类型

    1. d、i、u、x、o的整型数值，是指最小数字位数，不足的位要在左侧补0，如果超过也不截断，缺省值为1
    2. a、A、e、E、f、F的浮点数值，指小数点右边显示的数字位数，必要时四舍五入或补0；缺省值为6
    3. 对应于g、G的浮点数值，指有效数字的最大位数；缺省值为6
    4. 对应于s的字符串类型，是指输出的字节的上限，超出限制的其它字符将被截断
    5. 如果域宽为*，则由对应的函数参数的值为当前域宽。如果仅给出了小数点，则域宽为0

  • length

    指出浮点型参数或整型参数的长度

    1. hh对应于整数类型，printf期待一个从char提升到int尺寸的整型参数。
    2. h对应于整数类型，printf期待一个从short提升到int尺寸的整型参数。
    3. l
      • 对应于整数类型，printf期待一个long尺寸的整型参数。
      • 对应于浮点类型，printf期待一个double尺寸的整型参数.
      • 对应于字符串s类型，printf期待一个wchar_t指针参数。对于字符c类型，printf期待一个wint_t型的参数。
    4. ll对应于整数类型，printf期待一个long long尺寸的整型参数。Microsoft也可以使用I64。
    5. L对应于浮点类型，printf期待一个long double尺寸的整型参数。
    6. z对应于整数类型，printf期待一个size_t尺寸的整型参数。
    7. j对应于整数类型，printf期待一个intmax_t尺寸的整型参数。
    8. t对应于整数类型，printf期待一个ptrdiff_t尺寸的整型参数。

来个稍微麻烦点的看看：
printf("|%2$+10.5hhd|%1$c|\n", 'a', 'b');

类型标识字符

  • d，i有符号十进制数值int。%d与%i对于输出是同义；但对于scanf()输入二者不同，其中%i在输入值有前缀0x或0时，分别表示16进制或8进制的值。如果指定了精度，则输出的数字不足时在左侧补0。默认精度为1。精度为0且值为0，则输出为空。
  • u，十进制unsigned int。如果指定了精度，则输出的数字不足时在左侧补0。默认精度为1。精度为0且值为0，则输出为空。
  • f，F，double型输出10进制定点表示。%f与%F差异是表示无穷与NaN时，%f输出inf，infinity与nan，%F输出INF，INFINITY与NAN。小数点后的数字位数等于精度，最后一位数字四舍五入。精度默认为6。如果精度为0且没有#标记，则不出现小数点。小数点左侧至少一位数字。
  • e，E，double值，输出形式为10进制的([-]d.ddd e[+/-]ddd). E版本使用的指数符号为E（而不是e）。指数部分至少包含2位数字，如果值为0，则指数部分为00。小数点前存在1位数字。小数点后的数字位数等于精度。精度默认为6。如果精度为0且没有#标记，则不出现小数点。
  • g，G，double型数值，精度定义为全部有效数字位数。当指数部分在闭区间 [-4,5] 内，输出为定点形式；否则输出为指数浮点形式。%g使用小写字母，%G使用大写字母。小数点右侧的尾数0不被显示；显示小数点仅当输出的小数部分不为0。
  • x，X16进制unsigned int。%x使用小写字母；%X使用大写字母。如果指定了精度，则输出的数字不足时在左侧补0。默认精度为1。精度为0且值为0，则输出为空。
  • o，8进制unsigned int。如果指定了精度，则输出的数字不足时在左侧补0。默认精度为1。精度为0且值为0，则输出为空。
  • s，如果没有用l标志，输出null结尾字符串直到精度规定的上限；如果没有指定精度，则输出所有字节。如果用了l标志，则对应函数参数指向wchar_t型的数组，输出时把每个宽字符转化为多字节字符，相当于调用wcrtomb函数。
  • c，如果没有用l标志，把int参数转为unsigned char型输出；如果用了l标志，把wint_t参数转为包含两个元素的wchart_t数组，其中第一个元素包含要输出的字符，第二个元素为null宽字符。
  • p，void *型
  • a，A，double型的16进制表示，”[−]0xh.hhhh p±d”。其中指数部分为10进制表示的形式。例如：1025.010输出为0x1.004000p+10。%a使用小写字母，%A使用大写字母。
  • n，不输出字符，但是把已经成功输出的字符个数写入对应的整型指针参数所指的变量。
  • %，%字面值，不接受任何flags，width，precision和length。
