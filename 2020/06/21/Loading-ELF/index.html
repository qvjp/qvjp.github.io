这篇讲讲ELF文件的加载和执行。

ELF是什么

ELF（Executable and Linkable Format），可执行可链接文件。最初由Sun公司设计并应用在Unix V4上。具体格式规范可见System V ABI。由于ELF格式的通用性，目前被多种OS支持，可用于可执行文件、共享库文件、目标文件、和Core Dumps文件。

文件结构

ELF文件由ELF Header和其他的填充数据构成，这些填充数据又分为：

  • Program Header Table，用于描述0个或多个内存segment
  • Section Header Table，用于描述0个或多个section
  • 上两个表所指向的数据

Program Header

Program Header是下边结构的数组，具体数组个数由ELF Header中44-45位指出。

Position Field Value
0-3 p_type Type of segment (see below)
4-7 p_offset The offset in the file that the data for this segment can be found
8-11 p_vaddr Where you should start to put this segment in virtual memory
12-15 p_paddr Undefined for the System V ABI
16-19 p_filesz Size of the segment in the file
20-23 p_memsz Size of the segment in memory
24-27 p_flags Flags Flags: 1 = executable, 2 = writable, 4 = readable.
28-31 p_align The required alignment for this section (must be a power of 2)

其中第0-3位标识本Segment的类型，具体如下：

Value ISA
0x000 Program header table entry unused
0x001 Loadable segment
0x002 Dynamic linking information
0x003 Interpreter information
0x004 Auxiliary information
0x005 reserved
0x006 segment containing program header table itself
0x007 Thread-Local Storage template

加载ELF

ELF文件头中描述了整个文件的布局信息，加载器根据这个信息将ELF文件读取到内存中，具体步骤为：

  • 读取ELF Header，ELF Header总是在文件最开始的部分，它包含了剩下部分如何布局的信息
  • 通过ELF Header找到Program Header Table，这个Header数组指明数据段、代码段等的具体位置、可执行文件的结尾在哪等信息
  • 根据Program Header提供的信息，分配物理内存，然后将物理内存映射给连续的虚拟内存，将Program Header中p_offset所指位置的长度为p_filesz的内容复制到虚拟地址（也就是复制到了物理内存），然后取消这一此虚拟地址到物理地址的映射
  • 然后将包含可执行代码的物理地址映射到Program Header指明的虚拟地址p_paddr

执行ELF文件

通过上边的布局、加载，目前可执行代码已经加载到内存中了，且有了程序入口header->e_entry，只要和处理其他Process一样，初始化一个进程，将eip寄存器指向入口地址，将进程链接到进程列表即可。
